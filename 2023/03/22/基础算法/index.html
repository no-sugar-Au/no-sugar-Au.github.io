基础算法

Posted on 2023-03-22 Edited on 2023-11-02 In 算法

进行基础算法的练习，包括对数据结构的复习和掌握，大约有二三百道题，题目难度从易到难，但大部分比较基础，希望能够坚持学习，克服困难

会尽量写出解题的思路，有一个思考的过程

数组

二分查找

思路：二分查找需要注意边界问题

  • 定义区间为 [left,right]
  • 定义区间为 [left,right)

搜索插入位置

思路：大致还是二分查找的思路，但是存在找不到的情况，那么如果找不到时，就是 target 小于等于 left 或大于 right ，但如何知道没有查找到对应的 target ，用 flag 来确认是否找到

以上是我的思路，但是后面又看了题解才意识到，当 while 循环结束时，一定会有 left=right+1 ，所以直接如果没有查找到 target ，直接返回 left 即可

在排序数组中查找元素的第一个和最后一个位置

思路：二分查找到一个元素，然后向两边查找是否还有与 target 相同的元素，个人感觉这种方法最容易明白

x 的平方根

思路：这道题可能用于让你不用开根号函数求一个数的平方根，可以用二分的思想这里需要单独考虑0的情况，思路不难

移除元素

思路：所谓数组中删除元素，其实是覆盖的操作，运用双指针，快指针遍历数组，遇到不等于要删除的元素就将值赋给新数组，元素下标为慢指针的值

删除有序数组中的重复项

思路：这个题与上个题思路差不多，都是使用快慢两个指针，但是我个人感觉最不一样的地方在于上一道题是题目给了一个值，你直接拿来与数组中的数比较即可，但这个题你与自身比较是没有意义的，所以初始定义时，让 slow=0 而 sort=1 ，循环结束后 slow+1 才是最终要的值（用for和while循环都是可以的，上个题用的for循环，这里试一试while循环）

移动零

思路：还是快慢双指针，最后把数组后面的值全为0即可

有序数组的平方

思路：数组中可能是既有正数又有负数，负数平方之后变成正数，如果数值大的话需要排到右侧，所以运用双指针的方法，分别从数组头和尾遍历，对比平方之后的数值，将大的放在数组的右边

长度最小的子数组

思路：该题运用滑动窗口的思想，一开始看这道题直接想的就是两层for循环，但是有更好的思路，j 是滑动窗口的右边界，随着 j 的增大，一旦存在 sum 的值大于等于目标值 target ，则进入了 while 循环，在此循环中，首先是计算长度，然后在每一次计算的长度中取最小值，其次是滑动窗口的左边界需要向后移动，那么把滑动窗口中左边界的值减掉，再让 i 的值加一，就实现了滑动窗口左边界向右移动，最后的最后需要判断是否没有符合条件的子数组，没有则 return 0

水果成篮

思路：这道题我觉得是有一点困难的，主要难在出现第三种水果种类时如何进行滑动窗口的移动，整个代码中我个人认为最难理解的就是 n 这个值，也就是15-17行代码，将 r 的值赋给 n 到底是什么作用。什么时候要执行 n=r 这个操作呢，就是当水果编号发生改变时就执行，不管 fruits[r] 是否在 arr 数组中，n 的位置就是水果种类发生变化的位置，那么不难想出，当出现第三种水果，需要将第一种水果排除在外时，n 所在的位置就是新的滑动窗口的左边界，让 l=n ，也就实现了滑动窗口左边界的移动（n 这个地方真的想了好久）

螺旋矩阵 II

思路：这道题单纯需要思路的清晰，不能在循环的边界上搞混，坚持左闭右开的原则，最后需要判断一下 n 是否为奇数，是奇数的话矩阵最中间是一个数，不是一个圈，单独赋值

螺旋矩阵

思路：这个题与上个题最大的区别就是这个题中的矩阵长宽不相等，但是大体思路差不多，直接看代码

链表

链表的定义

移除链表元素

思路：在链表中删除头结点与其他节点操作不一样，需要考虑两种情况，可以定义一个虚拟的头结点 dummyHead ，令 dummyHead.next=head ，这样就可以直接判断链表中的每个值是否与目标值相等，这里定义一个 temp 用于迭代（个人感觉迭代的方法更容易理解，递归的方法有点不太明白）

设计链表

思路：链表的基本操作，之前数据结构课都写过，这里巩固复习，需要注意的一个地方是：查找元素时 i<=index ，添加和删除元素时记得是 i<index

反转链表

  • 双指针解法

思路：pre=null 即头结点 head 的前一位，然后随着 cur 向后移动，把 cur.next=pre ，就改变了指向，但是由于 cur.next 发生了改变，所以在改变之前需要一个 temp 记录原本的 cur.next ，这样在改变朝向之后才能保证 cur 可以继续向后移动，顺序不能搞混

  • 递归解法

思路：定义一个 reverse 函数，其中的逻辑其实与双指针解法相同，当 head===null 时递归结束，个人感觉还是双指针的写法更容易

两两交换链表中的节点

思路：首先设置一个虚拟头结点，举个例子，如果测试用例是 [1,2,3,4] ，那么从头开始时， tmp1 存储的是1， tmp2 存储的是3，过程是先让 cur 指向2，再让2指向1，但此时2指向1，已经找不到后续节点，这里 tmp2 的作用就体现出来，可以找到3的位置，然后 cur 的位置向后移动两位。循环的结束条件很重要，是考虑到节点数量分别为奇数和偶数时的情况，并且这里的两个条件顺序不能写反，否则如果 cur.next===null ，再判断 cur.next.next 就会出现空指针异常

删除链表的倒数第 N 个结点

思路：定义快慢指针，由于找的是倒数第n个结点，所以让快指针先走n步，然后快慢指针再同时走，当快指针走到最后一个非空结点时，慢指针正好走到倒数第n个节点的前一位

链表相交

思路：这里只看代码总感觉不好理解，我个人认为，举个例子来说，就是有 a 和 b 两个人在 headA 和 headB 两条跑道上跑，a 跑完了 headA 就去 headB 的起点接着跑，b 跑完了 headB 就去 headA 的起点接着跑，跑完了一条跑道就去另一个跑道的起点接着跑，这两条跑道从一个地方开始合并成一个跑道，当 a 和 b 两个人在跑道交汇的起点相遇时，就结束跑步，相反，只要不在这个地方相遇，就跑到相遇为止

环形链表 II

思路：定义快慢指针，快指针一次走两步，慢指针一次走一步，如果存在环，那么快慢指针一定会在环上的某一处相遇，通过等式代换可以求得从相遇点开始到环的起点也是终点的距离等于从链表头到环的起点的距离，所以再定义两个指针从这两个位置出发，相遇的位置就是环的起点（自己是真的想不出这么奇妙的方法，感谢卡尔大神）

两数之和

思路：遍历数组，定义 map 用来存放数组中的数，关键代码在4、5行，这里判断一下目标值 target 减去当前遍历到的数组元素得到的值是否存在于 map 中，即二者相减的数是否也是数组中的数，如果if判断成立，则出结果

四数相加II

思路：先定义 map ，第一个双循环计算出前两个数组中的数相加的和出现的次数，并存放在 map 中，第二个双循环计算后两个数组中的数相加的和出现的次数，如果第一个双循环中的sum和第二个双循环中的sum相加为0，那么该情况就满足条件，相当于如果 map 中存在值等于第二个双循环中的 0-sum 满足条件，此时用于计数的 count 就可以加上这种情况，累加的数值为满足条件时 map 中该值的出现次数

栈和队列

有效的括号

思路：首先判断字符串长度，如果为奇数则不用进行后续判断。然后给定 map 存储括号对应关系，可以理解为，右括号对应的是属性名，左括号对应的是属性值。遍历字符串，第一个if判断当前元素是否为右括号，如果为右括号，第二个if则判断如果此时已经栈空或者左右括号并不对应，则 return false ，否则将匹配上的左括号弹出栈顶。此时再回到第一个if判断，如果遍历到的元素不是右括号而是左括号，则将其压入栈中，继续下一次循环

删除最外层的括号

  • 方法1

思路：计数器的方法，遇到左括号-1，右括号+1，可以设想，如果不出现例如（（ 或者 ）） 的情况时，只有单纯的 （） 这种类型，那么计数器从0开始，也一定是以0结束，意思是只要左括号计数大于1的和右括号大于0的，就是添加到最终输出字符中的括号

  • 方法2

思路：借助栈实现，看代码就能够清晰看懂思路

删除字符串中的所有相邻重复项

思路：这个题比较简单，只需要判断元素是否与栈顶元素相同，相同则栈顶元素出栈，最后将栈return

逆波兰表达式求值

思路：遍历数组，是数字则进栈，是运算符号则出栈两个数字进行相应的运算，然后再将结果进栈。需要特殊注意的地方在于，题目中要求两个整数间的除法总是 向零截断 ，这里本来用的是 Math.floor(n2/n1) ，但二者取整有所区别，n2 / n1 | 0 的结果是一个整数，而 Math.floor(n2 / n1) 的结果是一个浮点数。位运算符 | 会将浮点数结果转换为整数，相当于取整操作。而 Math.floor() 函数会直接返回向下取整后的浮点数结果。在处理负数时，n2 / n1 | 0 中的位运算符 | 对负数进行取整时，会截断小数部分并向零方向取整。而 Math.floor(n2 / n1) 函数会向下取整，即总是向负无穷方向取整。

二叉树

二叉树的前序遍历

思路：递归遍历法，判断当前 root 是否为空，是则直接返回，否则将当前节点的值 root.val 添加到数组 res 中，表示先访问该节点，然后递归调用 dfs 函数，遍历左子树和右子树。中序和后序遍历类似，只需要改变函数中的顺序即可

二叉树的层序遍历

思路：res 用于存放遍历结果，queue 用于存放待遍历的节点。先将 root 根节点压入栈中，queue 负责存放的其实是当前层级待遍历的节点，所以有几个节点就循环几次，将节点的值存进 tmpQueue 数组中，如果当前节点还有左右节点，就将左右节点再放进队列中，进行下一轮循环。循环结束后将 tmpQueue 中的数添加到数组 res 中然后返回即可
